# PTA — Paktin Travel Agency

Internal flight ticket management system for tracking bookings, finances, and generating sales reports.

## Tech Stack

- **Framework**: Next.js 15 (App Router, React 19)
- **Language**: TypeScript (strict)
- **Database**: Supabase (PostgreSQL, free plan) — server-side only via service role key
- **Auth**: Custom JWT stored in httpOnly cookies — NOT Supabase Auth
- **UI**: MUI v7 + Emotion + Tailwind CSS v4
- **State**: Zustand (auth) + React useState (forms/UI)
- **Charts**: Recharts
- **Validation**: Zod (API routes)
- **Password hashing**: bcryptjs (cost 12)

## Database Tables

### users
Admin accounts only. Passwords stored as bcrypt hashes (cost 12).
Role: `admin` | `super_admin` — only super_admin can manage users.

### tickets
Core finance table. Fields: client info, from/to, dates, airline, prices.
- `due_amount` = `sold_price - paid_amount` — **calculated by BE, never set by FE**
- `commission` = `sold_price - original_price` — **calculated by BE, never set by FE**
- `ticket_number` — generated by BE: `PTA-{timestamp}-{random}`

### audit_logs
Every CREATE/UPDATE/DELETE/LOGIN/LOGOUT writes a row. Includes old_values, new_values, IP, user agent.

## Security

- httpOnly + Secure + SameSite=Strict cookies for JWT
- JWT expires in 8 hours
- Rate limiting: 5 failed login attempts → 30-min lockout per IP
- Middleware verifies JWT on every request; injects x-user-id, x-user-role headers
- All mutations validated with Zod schemas
- Passwords hashed with bcrypt cost 12
- Security headers on all responses (X-Frame-Options DENY, etc.)

## API Conventions

- All API routes live under `src/app/api/`
- Auth headers injected by middleware: `x-user-id`, `x-user-username`, `x-user-role`
- Responses: `{ data: T }` for success, `{ error: string }` for errors
- Paginated: `{ data: T[], total, page, pageSize }`
- Dashboard period values: `today | yesterday | this_week | this_month | this_year`

## Component Conventions (matches acs-web)

- Each component: `ComponentName/index.tsx` + `ComponentName/styles.ts`
- Styles use MUI `sx` prop objects exported from `styles.ts`
- Pages: `page.tsx` + `page.styles.ts` co-located
- `'use client'` directive only on components that need browser APIs or hooks
- Forms: controlled components with local useState, Zod validation before POST

## PWA

`public/manifest.json` + meta tags in root layout. Supports add-to-homescreen on mobile.

## Running Locally

```bash
cp .env.local.example .env.local
# Fill in Supabase URL, keys, and JWT_SECRET
npm install
# Run Supabase migrations in supabase/migrations/
npm run dev
```
